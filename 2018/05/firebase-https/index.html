    墙外使用 Firebase 搭建免费的 HTTPS 服务器

    2018-05-18
    • 开始前的必要条件
    • 在 firebase 中新建工程
    • 安装 firebase 并授权
    • 初始化 firebase 工程
    • 部署 firebase 工程

    当下很多流行的 web 技术都会强制要求使用HTTPS的服务器，自签名的证书无法部署到实际环境中进行测试，这在调测方面有一定局限性。本篇文章将会使用 Google 的 Firebase 技术实现免费的 HTTPS 服务器。

    如果没有翻墙条件，可以考虑使用 Netlify 的方式进行 HTTPS 服务器的部署，唯一缺点是需要将源码放到 GitHub 中，如果不在意可以进行尝试。

    开始前的必要条件

    • Node JS 安装，这个是标配，基本上现在 Web 技术离不开它
    • 由于使用 Google 产品，所以需要翻墙，墙外的风光无限好
    • 准备一个标准的 web 工程

    在 firebase 中新建工程

    在 firebase console 中创建一个工程，记住工程名称，如这里创建一个 pwademo 的工程

    安装 firebase 并授权

    firebase-tools 是一个标准的 cli 终端工具，需要使用 npm 进行安装。安装完成后，第一次使用 firebase 则需要使用 firebase login 进行授权登录。

    $ npm install -g firebase-tools
$ firebase login

    初始化 firebase 工程

    进入本地项目的根路径，使用 firebase init 命令进行初始化操作。期间会询问一些问题，如 index.html 所在位置，是否覆盖等等，根据自己的项目结构做出应答即可。

    $ cd pwademo
$ firebase init

    在使用 firebase init 之后会在当前目录下生成 .firebaserc 和 firebase.json 两个文件，这两个文件关联了创建的 firebase 的环境。

    部署 firebase 工程

    最后在根路径下执行 firebase deploy 即可部署到 Google 的服务器中，并会在终端中输出可访问的 HTTPS 地址。

    $ firebase deploy
