    在 ionic 中配置 material 主题风格

    2018-05-13
    • 安装material依赖

    • 重写复制任务配置
    • 首页配置关联样式

    ionic 本身具有丰富的标准组件库，material 是一种从色彩搭配到组件布局的设计风格，具有开源实现。如果你不满足于 ionic 标准的组件库，可以结合 material 组件搭配出更加漂亮的设计。

    本篇文章将详细说明 ionic 如何集成 material 风格样式。

    安装material依赖

    本文假设你已经有一个标准的 ionic 工程。

    进入 ionic 工程根路径，执行以下命令进行 material 的安装。

    $ npm install --save @angular/material @angular/cdk @angular/animations

    这种安装方式会存在一个隐性的问题，就是当前环境中 angular 和 material 不是同一个版本，这样在运行时会出现 “Type ‘ElementRef’ is not generic” 的错误，执行完上述命令后，应该检查下 package.json 文件，下面的依赖版本是否一致。

    "dependencies": {
  "@angular/core": "^5.2.0",
  "@angular/cdk": "^5.2.0",
  "@angular/material": "^5.2.0"
}

    当然你也可以直接按上面的版本号在 package.json 中配置好，然后执行 npm install 进行安装。

    导入material配置

    打开 src/app/app.module.ts 文件，导入 BrowserAnimationsModule 模块

    import { BrowserAnimationsModule } from '@angular/platform-browser/animations';

@NgModule({
  ...
  imports: [
    ...
    BrowserAnimationsModule
  ],
  ...
})

    重写复制任务配置

    编辑 package.json 文件，在底部增加如下内容：

    "config": {
  "ionic_copy": "./config/copy.config.js"
}

    之后在根路径创建 ./config/copy.config.js 文件，内容如下：

    module.exports = {
  copyAssets: {
    src: ['/assets/**/*'],
    dest: '/assets'
  },
  copyIndexContent: {
    src: ['/index.html', '/manifest.json', '/service-worker.js'],
    dest: ''
  },
  copyFonts: {
    src: ['/node_modules/ionicons/dist/fonts/**/*', '/node_modules/ionic-angular/fonts/**/*'],
    dest: '/assets/fonts'
  },
  copyPolyfills: {
    src: [`/node_modules/ionic-angular/polyfills/${process.env.IONIC_POLYFILL_FILE_NAME}`],
    dest: ''
  },
  copySwToolbox: {
    src: ['/node_modules/sw-toolbox/sw-toolbox.js'],
    dest: ''
  },
  copyMaterialTheme: {
    src: ['/node_modules/@angular/material/prebuilt-themes/indigo-pink.css'],
    dest: '/assets'
  }
}

    这个文件的作用是，当在执行 ionic build 或者 ionic serve 等构建任务时，自动将资源文件进行拷贝，这样在 www/ 文件夹中才会有样式、字体等文件。

    copy.config.js 的内容可以在 ./node_modules/@ionic/app-scripts/config/copy.config.js 中进行复制，其中 copyMaterialTheme 是我们增加的任务，其余的都是缺省的任务。

    material 为我们提供了不少色彩搭配的主题风格，从上述的 copyMaterialTheme 可以看出，我们使用了它的 indigo-pink.css 主题风格，当然除了这个还有其他的，具体可看上面配置路径下的文件。

    首页配置关联样式

    打开 src/index.html 文件增加上述配置的主题样式（indigo-pink.css）。在编译或者构建时会将 material 主题拷贝到 www/assets 目录中，这个在上面的 copyMaterialTheme 任务中已经说明，下面就是加载这样样式文件。

    <link href="assets/indigo-pink.css" rel="stylesheet">

    使用Material组件

    在 material conponents 中找到需要的组件，这里以 button 和 select 举例。

    在 app.module.ts 中导入组件依赖，如下：

    import { MatButtonModule } from '@angular/material/button';
import { MatSelectModule } from '@angular/material/select';

@NgModule({
  ...
  imports: [
    ...
    MatButtonModule,
    MatSelectModule
  ],
  ...
})

    从上面可以看出，没用一个 material 组件都需要在 app.module.ts 中进行导入，最后导致这个文件过于庞大不易阅读，更好的做法是在 src/app/ 下创建一个 material.module.ts 的文件，将 material 组件配置放在这里，如下：

    import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';

// material components
import { MatButtonModule } from '@angular/material/button';
import { MatSelectModule } from '@angular/material/select';

@NgModule({
    imports: [MatButtonModule, MatSelectModule],
    exports: [MatButtonModule, MatSelectModule]
})

export class MaterialModule {  }

    有了 material.module.ts 这个模块后，以后所有 material 组件都在这个文件中注册。在 app.module.ts 中就会干净许多，只需要将 material.module.ts 导入即可。

    import { MaterialModule } from './material.module';

@NgModule({
  ...
  imports: [
    ...
    MaterialModule,
  ],
  ...
})

    在页面中使用组件

    打开页面 html 文件，直接使用 material 标签的控件，如下：

    <button mat-raised-button>Basic</button>
<button mat-raised-button color="primary">Primary</button>
<button mat-raised-button color="accent">Accent</button>
<button mat-raised-button color="warn">Warn</button>
<button mat-raised-button disabled>Disabled</button>
<a mat-raised-button routerLink=".">Link</a>

    更多的组件使用还请看 material conponents 。

    运行 ionic serve -c 后结果为：

    使用Material图标

    $ npm install material-design-icons --save

    在 app.module.ts 中进行导入

    import { MatIconModule } from "@angular/material/icon";

@NgModule({
  ...
  imports: [
    ...
    MatIconModule
  ],
  ...
})
