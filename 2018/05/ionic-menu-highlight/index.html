    让 ionic 中 sidemenu 选中的菜单高亮显示

    2018-05-13
    • 工程准备
    • 添加高亮样式
    • 添加菜单激活事件
    • 实现菜单激活事件
    • 最后的完成效果

    ionic 的 sidemenu 用于菜单控件，默认的 sidemenu 在选中时无法高亮显示，当再次展开菜单时，无法明确的知道现处于哪个菜单之下，本篇文章就是解决此问题。

    工程准备

    现在默认已经有 sidemenu 布局的 ionic 工程，如果没有的话，可以使用 ionic start myapp sidemenu 进行创建，运行后如下，默认菜单选中不会高亮显示。

    添加高亮样式

    编辑 src/app/app.scss 样式文件，添加选中时的高亮样式

    .activeHighLight {
  background-color: map-get($colors, primary);
}

    这里使用默认定义在 variables.scss 中的 primary 颜色来定义选中时的菜单色

    添加菜单激活事件

    编辑 src/app/app.html 文件，在 button 标签中增加 [class.activeHighLight]=”checkActive(p)” 事件

    <ion-content>
  <ion-list>
    <button menuClose ion-item *ngFor="let p of pages"
        [class.activeHighLight]="checkActive(p)" 
        (click)="openPage(p)">
        
    </button>
  </ion-list>
</ion-content>

    实现菜单激活事件

    编辑 src/app/app.component.ts 文件，增加如下注释内容

    export class MyApp {
  // ....
  
  activePage: any; // 添加活动页属性

  constructor(public platform: Platform, public statusBar: StatusBar, public splashScreen: SplashScreen) {
    // ...

    // 缺省第一页为活动页面
    this.activePage = this.pages[0];
  }

  openPage(page) {
    this.nav.setRoot(page.component);
    
    // 跳转页面时赋值
    this.activePage = page;
  }

  // 添加激活样式的事件
  checkActive(page) {
    return page == this.activePage;
  }
}
