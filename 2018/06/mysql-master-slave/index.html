    两台机器分别安装好 mysql，本文以 mysql5.7 版本为例，安装可参考 centos7 中 mysql 的安装与配置 一文。机器环境是，192.168.1.34（master），192.168.1.35（slave）。

    1. 主从模式

    1.1 master配置

    1.1.1 文件配置

    编辑 vi /etc/my.cnf 的mysql配置文件，修改或增加如下配置：

    [mysqld]
server-id=1
# 没有就增加，有的是注释状态，取消掉注释，有的只有log-bin，没有路径，保持缺省即可
log-bin=/var/log/mysql/mysql-bin.log
# 忽略的数据库
binlog-ignore-db=mysql
binlog-ignore-db=sys
binlog-ignore-db=information_schema
binlog-ignore-db=performance_schema

# binlog-do-db=pets # 需要同步的数据库
# binlog_format=mixed # 主从复制的格式(mixed,statement,row,默认格式是statement)
# expire_logs_days=7 # 二进制日志自动删除/过期的天数。默认值为 0,表示不自动删除
# slave_skip_errors=1062 # 跳过主从复制中遇到的所有错误或指定类型的错误,避免 slave 端复制中断
# relay_log=edu-mysql-relay-bin # 从服务器时的中继日志
# log_slave_updates=1 # 表示slave将复制事件写进自己的二进制日志

    1.1.2 创建用户

    mysql> create user 'repl'@'%'  identified by '1q@W3e$R';
mysql> grant replication slave on *.*  to 'repl'@'%';

    1.1.3 确保连接有效性

    用户创建好后，可以在 slave 或者别的机器使用此用户远程登录一下，确保创建的用户能正常登录

    $ mysql -h 192.168.1.34 -u repl -p

    若远程登录失败或者出现拒绝等信息，应该检查几个地方：

    • 配置文件的bind-address
    • 防火墙的出入站规则
    • 新建的用户是否具有访问权限，IP或数据库是否有限制

    1.1.4 master状态

    mysql> show master status;

    1.2 slave配置

    1.2.1 文件配置

    编辑 vi /etc/my.cnf 的mysql配置文件，修改或增加如下配置：

    [mysqld]
server-id=2
# read-only=on # 设置从数据库只读，不能进行更新操作

    1.2.2 配置连接master

    mysql> CHANGE  MASTER  TO  MASTER_HOST='192.168.1.34',  MASTER_USER='repl',  MASTER_PASSWORD='1q@W3e$R',  MASTER_LOG_FILE='first-bin.000022', MASTER_LOG_POS=154;

    这里的 master_log_file 表示 master 中 binlog 的文件名称，master_log_pos 表示从哪个位置点开始同步，这些信息可以在上图看到。

    1.2.3 启动slave模式

    mysql> start slave;

    1.2.4 slave状态

    mysql> show slave status\G;

    2. 主主模式

    通常主主模式需要搭载高可用的软件，如keepalive，主主并不是就是两台，也会有从。同时主主代表的意思并不是两台同时提供服务，只是为了高可用性，当一台宕机的时候，另一台可以接管。

    基于测试的目的，主主以两台为基准测试。

    2.1 模式特点

    • 两台都可读写，互为主备，默认只使用一台读写，另一台备用
    • 两台主库之间互为高可用，可结合keepalive

    2.2 实现原理

    先搭建一主一从，再反向一次即可。这里需要注意的是ID自增长的问题，为了保证切换数据库后插入数据时ID不会冲突，一般两台是奇偶交替。下面的步骤是按照 主从模式 之后进行。

    2.3 slave配置

    [mysqld]
server-id=2
log-bin
binlog-ignore-db=mysql
binlog-ignore-db=sys
binlog-ignore-db=information_schema
binlog-ignore-db=performance_schema
binlog-do-db=pets

# 主键自增规则，主要解决ID自增长冲突的问题
auto_increment_increment=2 # mysql服务器数量，10台备份就写10，这里是2台
auto-increment-offset=2 # 这台服务器的序号，从1开始，不超过auto-increment-increment

    2.4 master配置

    [mysqld]
server-id=1
log-bin
binlog-ignore-db=mysql
binlog-ignore-db=sys
binlog-ignore-db=information_schema
binlog-ignore-db=performance_schema
binlog-do-db=pets

# 主键自增规则，主要解决ID自增长冲突的问题
auto_increment_increment=2 # mysql服务器数量，10台备份就写10，这里是2台
auto-increment-offset=1 # 这台服务器的序号，从1开始，不超过auto-increment-increment
